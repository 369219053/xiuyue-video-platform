# Excel 数据整理提示词

> 将此提示词发送给大模型，可让其自动理解甲方 Excel 结构并将数据整理为标准 JSON 格式。

---

## 提示词正文

```
你是一个数据处理专家。我有一个甲方提供的 Excel 文件，需要你按照以下规则将其整理成结构化数据。

---

## Excel 文件结构

该 Excel 包含多个 Sheet：

### 1. 汇总 Sheet（名称：ai数量需求汇总）
记录每个组、每个产品需要生成的视频数量。结构如下：

| 日期 | 组 | 品类 | 需求数量 |
|------|-----|------|---------|
| 5.16 | 何欢组 | redhalo生物素 | 40 |
|      |       | 予香琳喷雾 | 40 |
|      | 曹心唯组 | 缮丝源乌发液 | 40 |

注意："组"列存在合并单元格，空白时继承上一行的值。

### 2. 各组 Sheet（如：何欢组、曹心唯组 等）
每个 Sheet 内包含一个或多个产品段落，每个产品段落结构固定为：

**第一行：产品标题行**
- 仅第一列有内容，为产品名称（含括号说明，如"缮丝源乌发液（需要加字幕）"）
- 其余列为空

**第二行：列标题行（表头）**
- 第一列为"日期"（或"hu"等等效标识）
- 后续列为字段名，如：人物a、人物b、场景、道具、台词、字幕、动作、镜头、视频比例等

**第三行起：数据行（模版）**
- 第一列为日期（如 5.15、5.16，以浮点数存储）
- 其余列为对应字段内容，部分列可能为空（用"/"表示）

---

## 处理规则

### Step 1：读取需求数量
从"ai数量需求汇总"Sheet 解析出每个组每个产品的目标数量：
{
  "何欢组": [("redhalo生物素", 40), ("予香琳喷雾", 40)],
  "曹心唯组": [("缮丝源乌发液", 40)],
  ...
}

### Step 2：扫描各组 Sheet，提取产品段落
识别规则：
- 产品标题行：第一列为非空字符串、非数字、非日期，且其余列基本为空（非空列 ≤ 2）
- 表头行：第一列为"日期"或"hu"，或行内含"人物a"、"场景"、"道具"等字样
- 数据行：第一列为浮点数日期（如 5.15）

### Step 3：匹配产品段落
将汇总表中的产品名与 Sheet 中的产品标题做模糊匹配：
- 去掉括号内容后比较
- 不区分大小写、忽略空格
- 无精确匹配时，兜底使用该 Sheet 的第一个产品段落

### Step 4：循环补齐数量
模版数据行数通常少于需求数量，需循环复用：
- 如模版有 20 条，需求 40 条 → 将 20 条循环两次
- 如模版有 15 条，需求 40 条 → 循环后截取前 40 条

### Step 5：输出格式
每个产品输出一个 JSON 数组，结构如下：
[
  "产品标题（含括号说明）",
  "列1 // 列2 // 列3 // ...",
  "数据1列1 // 数据1列2 // ...",
  "数据2列1 // 数据2列2 // ...",
  ...
]

格式规范：
- 第 1 个元素：产品标题（原始文本，保留括号）
- 第 2 个元素：所有列名用" // "拼接
- 第 3 个元素起：每条数据行，各列值用" // "拼接，空值用"/"替代
- 行尾多余的空列可省略

---

## 示例输出

[
  "缮丝源乌发液（需要加字幕）",
  "日期 // 人物a // 人物b // 场景 // 道具 // 台词 // 字幕 // 动作 // 镜头 // 视频比例",

  ...
]

---

请按照以上规则处理我提供的 Excel 数据。
```

## 补充说明

### 已知的表格兼容性问题

| 问题 | 表现 | 处理方式 |
|------|------|---------|
| 表头第一列非标准 | 李文晴组用"hu"代替"日期" | 同时识别"hu"和"日期"，或检测行内含"人物a"等关键词 |
| 合并单元格 | 汇总表"组"列有合并单元格 | 解析时继承上一行非空值 |
| 产品名大小写不一致 | 需求表"Freehalo"与标题"freehalo"不同 | 模糊匹配时统一转小写 |
| 产品段落少于需求数量 | 模版只有 4 条但需要 20 条 | 循环复用模版行，截取到精确数量 |
| 一个 Sheet 多个产品 | 燕娜组含两个产品段落 | 按产品标题行分割，各自独立匹配 |
