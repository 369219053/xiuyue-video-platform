# 代码清洗规则

本文档梳理从「甲方钉钉文档 / Excel 副表」→「发送给 Coze 工作流的最终数据行」全过程中代码做的所有清洗、匹配、组合逻辑。便于核对每一步是否符合预期。

涉及核心文件：
- `src/dingtalk_client.py`（钉钉文档拉取、产品段落扫描、日期/段落匹配工具）
- `src/excel_parser.py`（段落识别、列分组、随机组合 `_random_combine`，钉钉模式也复用这里）
- `src/app_server.py` → `expand_tasks`（把任务列表展开为最终数据行的总调度）

---

## 一、整体流程概览

```
甲方钉钉文档 / 上传的 Excel
        │
        ▼
[1] 拉取原始数据并转成二维行矩阵
        │   钉钉：dingtalk_doc_to_sheets → _sheet_to_rows（稀疏 JSON → 行）
        │   Excel：openpyxl.iter_rows
        ▼
[2] 单元格清洗  _clean_cell
        │   None / 空 → '/'
        │   时间型小数(0~1)还原比例 "9:16"
        │   换行符 → 空格
        ▼
[3] 扫描产品段落  _scan_sections
        │   识别「产品标题行 → 表头行 → 模板数据行」三段式结构
        │   过滤"空壳行"（只有日期/比例的占位行）
        ▼
[4] 汇总表解析  parse_mode_map + parse_demand_map
        │   读 ai数量需求汇总，得到 (组, 产品, 日期) → 模式 / 需求数量
        ▼
[5] 任务展开  expand_tasks（核心调度）
        │   对每个 (sheet, 产品, 任务日, 数量, 模式) 走匹配 + 取数 + 组合
        ▼
[6] 段落匹配  find_section
        │   同一产品多段 → 合并所有模板行 + 日期池为虚拟段落
        ▼
[7] 模式分流：模式 1 / 模式 2
        │
        ▼
[8] 按日期筛选模板  filter_templates_by_date
        │   按表头里的"日期"列精确取该日期的模板行
        ▼
[9] 组合生成  _random_combine
        │   视觉组锁定 + 台词去重队列 + 颜色洗牌 + 时长兜底
        ▼
[10] 列数对齐 → 发送给 Coze 的 input 数组
```

## 二、单元格清洗（`_clean_cell`）

| 输入类型 | 处理 | 示例 |
|---------|------|------|
| `None` / 空字符串 | 输出 `'/'`（占位符）| `None` → `'/'` |
| `datetime.time` | 转 `H:MM` | `09:16:00` → `'9:16'` |
| `float ∈ (0, 1)` 且 `len(repr) > 5` | 视为时间小数，还原为 `H:M` 比例 | `0.386111111…` → `'9:16'` |
| 其他 | `str(cell).strip()`，换行符替换为空格 | `'A\nB'` → `'A B'` |

> 这是 Excel/钉钉自动把 "9:16" 类比例识别成时间的坑，专门做了启发式还原。

---

## 三、产品段落识别（`_scan_sections`）

把一个 Tab 内的行切成多个段落，每段就是一个"产品块"。

**三段式结构**：
1. **标题行** `_is_title_row`：首列是非空字符串、非日期、其余列基本为空（≤2 列有值）
2. **表头行** `_is_header_row`：首列是 `日期` / `hu`，或行内含 `人物a / 场景 / 道具 / 输入指令台词` 关键词
3. **数据行**：模板行，**必须 ≥2 列有实质内容（非 `/` 非空）** 才计入，否则丢弃

**段落结构**：
```python
{
  'title':       '产品名（可能带括号备注）',
  'header_str':  '日期 // 人物a // 人物b // 场景 // ...',  # ' // ' 拼接
  'col_count':   <表头列数>,
  'templates':   ['5.16 // 50岁女性 // ... // 9:16', ...],  # 每行同样 ' // ' 拼接
  'available_dates': ['5.16', '5.23', ...],  # 该段日期列出现过的所有日期（去重保序）
}
```

---

## 四、日期归一（`_normalize_date` + `_date_key`）

钉钉里日期可能是：`float(5.16)`、`int(42510)`（Excel 序列号）、`str('5月16日' / '5/16' / '5.21.')`。

| 输入 | 输出 |
|------|------|
| `5.16`（float）| `'5.16'` |
| `42510`（Excel 序列号）| `'5.16'`（按 1900 历法换算）|
| `'5月16日'` / `'5月16号'` | `'5.16'` |
| `'5/16'` / `'5-16'` | `'5.16'` |
| `'5.21.'` | `'5.21'`（剥末尾点）|

**`_date_key` 二次归一**：把 `'5.20'` 和 `'5.2'` 统一为同一个键（解决钉钉 float 把 `5.20` 存成 `5.2` 的丢精度问题）。

---

## 五、产品段落匹配（`find_section`）

按 `(sheet, product_name)` 找产品段落。匹配规则：
- `sheet` 严格相等
- `product_name` 去括号备注 + 转小写 + 去空白

**如果同一产品在 Tab 内有多个段落**（甲方常按日期分多段写）：
- 把所有段落的 `templates` 和 `available_dates` **合并成一个虚拟段落**
- 这样后续 `filter_templates_by_date` 才能按日期列正确筛选，不依赖段落在文档里的位置先后
- 控制台打印 `ℹ️  「sheet·product」找到 N 个同名段落，合并所有模板行后按日期筛选`

> Excel 模式的 `_match_section` 走另一套：取**最后一个**同名段落（旧在上、新在下的约定）。这是因为 Excel 副表里同一产品通常只有"加字幕/不加字幕"两份完整模板，不是按日期分段。



---

## 六、按日期筛选（`filter_templates_by_date`）

输入：段落 `sec`、目标日期（如 `'5.23'`）。

1. 找表头里的 `"日期"` 列下标 → `date_col`
2. 用 `_date_key` 把目标日期归一为 key（如 `'5.23'` → `'5.23'`）
3. 遍历段落所有模板行，取该行第 `date_col` 列的值同样归一后与目标 key 相等的行
4. **如果段落没有"日期"列** → 直接返回全部模板（说明该段落不按日期组织）

---

## 七、找最近日期（`find_recent_dates`）

输入：可用日期池 `avail`、目标日期 `target`、需要的个数 `n`。

1. 过滤出**所有 ≤ target** 的日期
2. 按时间**降序**排序（最新→最旧）
3. 去重后取前 `n` 个

| 目标日 | 可用池 | n=1 | n=2 |
|--------|--------|-----|-----|
| 5.23 | [5.16, 5.23] | `['5.23']` | `['5.23', '5.16']` |
| 5.23 | [5.16, 5.20] | `['5.20']` | `['5.20', '5.16']` |
| 5.15 | [5.16, 5.23] | `[]`（无 ≤5.15 的）| `[]` |

> 任务日本身在池里也算"最近一个"（不强制要求当天有模板，因为当天就是最近）。

---

## 八、模式 1 取数逻辑

**目标**：用 ≤ 任务日最近那 1 天的模板，全部由这天的数据生成。

```
[1] recent = find_recent_dates(avail, date, n=1)
    若空 → ❌ "无任何可用资料日期，跳过"

[2] d1 = recent[0]
    pool_1 = filter_templates_by_date(sec, d1)
    若空 → ❌ "{d1} 模板为空，跳过"

[3] 若 d1 != 任务日 → ⚠️ "当天无资料，使用最近日期 {d1}"

[4] 取数：
    if len(pool_1) >= count:
        rows = pool_1[:count]              # 直接取前 N 条，不打乱不重组
    else:
        rows = _random_combine(pool_1, header_str, count)  # 不够则随机组合
```

> ⚠️ 直接取前 N 条是因为：池子够大时按原始顺序更可控；不够时才进入随机组合逻辑补齐。

---

## 九、模式 2 取数逻辑

**目标**：用 ≤ 任务日最近 2 天的模板**各占一半**混用，增加多样性。

```
[1] recent = find_recent_dates(avail, date, n=2)
    若空 → ❌ "无任何 ≤ 该日期的模板，跳过"

[2] 只找到 1 个日期 → ⚠️ "仅找到 1 个可用日期，全部由该日期生成"
    rows = _random_combine(pool_only, header_str, count)

[3] 找到 2 个：d_new = recent[0], d_old = recent[1]
    pool_new = filter_templates_by_date(sec, d_new)
    pool_old = filter_templates_by_date(sec, d_old)

[4] 均分：
    n_new = count // 2           # 新日期占一半（向下取整）
    n_old = count - n_new        # 旧日期占剩余
    rows = _random_combine(pool_new, ..., n_new) + _random_combine(pool_old, ..., n_old)
    ℹ️ "模式2 混用 d_new(n_new 条) + d_old(n_old 条)"

[5] 其中一个池为空 → 全部由另一个池生成，附 ⚠️ 警告
```

| 任务 | count | n_new | n_old |
|------|-------|-------|-------|
| 10 条 | 10 | 5 | 5 |
| 11 条 | 11 | 5 | 6 |
| 3 条 | 3 | 1 | 2 |

## 十、随机组合（`_random_combine`）— 最核心的清洗算法

把候选模板池 + 表头 + 需求数量 → 组合出 `count` 条新数据行。

### 10.1 列分组（`_classify_columns`）

按表头关键词把每一列归到三组之一：

| 分组 | 关键词命中 | 行为 |
|------|----------|------|
| **台词组** `text_group` | 列名含 `台词` 或 `字幕` | 整组锁定到同一原始行 |
| **视觉组** `visual_group` | 列名含 `人物 / 角色 / 场景 / 道具 / 动作 / 镜头 / 参考图 / 环境 / 背景 / 时长` | 整组锁定到同一原始行 |
| **人物子集** `char_group` | 含 `人物 / 角色` 的列 | 视觉组的子集，用于颜色洗牌 |
| **独立列** | 上面都不命中（如 `日期`） | 每列独立随机抽 |

> **核心理念**：「人物 + 场景 + 道具 + 动作 + 镜头 + 时长」整组绑定 → 不会出现"睡衣 + 医生办公室"这种乱搭；「台词 + 字幕」整组绑定 → 字幕和台词永远配对。

### 10.2 台词去重队列

为了让生成的前 K 条数据**每条台词都不同**（K = 模板里唯一台词数）：

1. 只按"含台词且不含字幕"的列做去重 key（字幕是台词的纠错版，不算独立内容）
2. 去重后所有唯一台词行 → 打乱 → 入队列
3. 生成时先从队列里 `pop(0)` 取，队列耗尽后再从所有 text_rows 随机抽

### 10.3 颜色洗牌（`_swap_colors_in_text`）

只在**人物 a/b 列**做服装颜色随机替换，增加视觉多样性。

- **颜色池**：从原模板所有人物列里扫描出现过的"颜色 + 服装词"组合，去重保序
- **替换规则**：每个匹配位置随机替换为颜色池中**不同的**颜色（避免无效替换）
- **黑名单保护**：`白大褂` 等固定搭配不替换
- **跨字符识别**：`蓝色polo领衬衫` 这种"颜色色 + 修饰词 + 服装词"也能识别（但必须有"色"字才允许跨字符）

服装词覆盖：`衬衫 / 上衣 / T恤 / 短袖 / 长袖 / 外套 / 马甲 / 大衣 / 风衣 / 夹克 / 卫衣 / 毛衣 / 裙 / 裤 / 西装 / 套装 / 工装 / 制服 / 护士服 / 中医服 / 大褂 / 长袍 / 长衫 / 中山装 / 唐装 / 旗袍 / polo / 围裙 / 围巾 / 帽子 / 眼镜 / 墨镜 / 框 / ...`

### 10.4 时长兜底（`_fix_duration_value`）

`时长` 列单元格：
- 空 / `/` 占位 / 非纯数字 → 回退为 `'10'`（默认值）
- 已是合法数字（如 `9:16`、`10`、`15.5`）→ 原样保留

### 10.5 单条生成流程

每条新数据按这个顺序填字段：

```
new_parts = ['/' * col_count]

[1] 视觉组：rng.choice(visual_rows) → 整组列填入
[2] 台词组：text_queue.pop(0) 或 rng.choice(text_rows) → 整组列填入
[3] 独立列：rng.choice(columns[i]) → 单列随机
[4] 颜色洗牌：对人物列做颜色替换
[5] 时长兜底：时长列空值补 '10'

result.append(' // '.join(new_parts))
```

## 十一、列数对齐

`_random_combine` 输出的每行字符串送给 `expand_tasks` 后还会做一次列数对齐：

```python
rows = [r + ['/'] * (col_count - len(r))  if len(r) < col_count
        else r[:col_count]
        for r in rows]
```

- 不足 `col_count` 列 → 用 `'/'` 补齐
- 超过 `col_count` 列 → 截断

最终每行的列数 = 表头列数，发给 Coze 工作流时字段不会错位。

---

## 十二、数据分隔符约定

全链路统一使用 `' // '`（**空格 + 双斜杠 + 空格**）作为字段分隔符：

```
日期 // 人物a // 人物b // 场景 // 道具 // 视频中人物说的台词 // 最终字幕的台词 // 人物动作 // 镜头 // 时长
```

发给 Coze 工作流的最终入参 `input` 是一个字符串数组，每个字符串是一行用 ` // ` 拼接的数据。

---

## 十三、警告与异常一览

`expand_tasks` 会把所有非致命问题收集到 `warnings` 数组返回给前端：

| 警告级别 | 触发条件 | 含义 |
|---------|----------|------|
| ❌ | 段落找不到 / 任何可用日期都没有 / 选中日期池为空 | 该任务被跳过，无数据输出 |
| ⚠️ | 模式 1 当天无资料用最近日期 / 模式 2 只找到 1 个日期 / 某一日期池为空 | 已生成数据，但用的不是预期日期 |
| ℹ️ | 模式 2 正常混用（"d_new(N条) + d_old(N条)"）| 仅信息提示 |

---

## 十四、检查清单（用于人工核对取数对不对）

发现某条数据可疑时，按这个顺序排查：

1. **看任务的模式**：`mode_map` 里 `(组, 产品, 任务日)` → 模式 1 还是 2？
2. **看产品段落是否合并**：控制台是否打印过 `ℹ️  找到 N 个同名段落，合并所有模板行`？
3. **看 `available_dates`**：合并后该产品的可用日期池是哪几个？
4. **算出实际取了哪个日期**：
   - 模式 1：`find_recent_dates(avail, 任务日, n=1)` → `recent[0]`
   - 模式 2：`find_recent_dates(avail, 任务日, n=2)` → `recent[0]` 和 `recent[1]`
5. **筛出那个日期的模板池**：`filter_templates_by_date(sec, d)` 应该返回的行
6. **判断是直接取前 N 条还是 `_random_combine`**：池子条数 ≥ 需求数 → 直接取前 N（模式 1 特有）
7. **如果走 `_random_combine`**：核对视觉组/台词组的"整组绑定"是否被破坏，颜色是否被洗牌

如果以上每一步预期值与实际不符，就是 bug；如果每一步对得上但结果仍不符合预期，则是规则本身需要调整。
