## 三、废片重跑机制

### 第一期方案（Excel 标记）

```
① 我们将 Sheet2「成品清单」发给甲方
         ↓
② 甲方审核视频，在「废片标记」列填 Y
         ↓
③ 甲方将标记好的 Excel 发回给我们
         ↓
④ 放入指定目录，运行命令：
   python main.py --retry data/output/台账/20250521_处理台账.xlsx
         ↓
⑤ 脚本自动读取所有标记 Y 的条目
   → 查找原始数据
   → 从失败/废片阶段重新调用工作流
   → 下载新视频，覆盖旧文件
   → 更新台账
```

### 第二期方案（后期升级，Web 表单）
- 在服务器上搭建简单 Web 界面
- 甲方在网页上查看视频、勾选废片
- 后台自动触发重跑，无需人工介入

## 四、交付时间线

```
D+0 上午：甲方发来 Excel
D+0 上午：运行脚本，开始处理
D+0 下午：工作流1全部完成（约10分钟）
D+0 下午~D+1：工作流2系列处理中
D+1 晚上：全部完成，整理台账，下载视频，交付甲方
```
