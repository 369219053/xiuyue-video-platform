## 一、当前实际架构（一键自动化阶段）

```
甲方 Excel
    │
    ▼  [界面上传]
python3 src/app_server.py（PyWebView 桌面窗口）
    │  在界面解析 Excel → 按 日期/组名/产品 展示
    │  界面填写：Coze Token、工作流ID、飞书AppID/Secret、BaseToken
    │
    │  [点击「⚡ 确认，开始执行」]
    ▼
Step 1/3：飞书 CDP 建副表
    │  通过 Chrome DevTools Protocol（CDP）操控浏览器
    │  自动在飞书多维表格创建副表并写入数据
    │  失败只打警告，不中断后续步骤
    ▼
Step 2/3：并行触发 Coze 工作流
    │  ThreadPoolExecutor（max_workers=5）并行调用
    │  入参：input（数据数组）、name（副表名）
    │  出参：debug_url（工作流执行页面链接）
    │  使用系统 curl，超时 120 秒，绕过 macOS LibreSSL 限制
    ▼
Step 3/3：写入飞书台账
    │  用飞书 App ID/Secret 换取 tenant_access_token
    │  调用 batch_create 接口写入记录
    │  台账地址：PSgJbhr21aHRGosg4uHclshgnDe / tblZrh3e6CWvvSwp
    │  字段：任务名称（日期_组名_产品）、debugurl
    ▼
data/user_state.json（配置 + 解析数据持久化，刷新不丢失）
```

## 二、模块职责

| 模块文件 | 职责 |
|----------|------|
| `src/app_server.py` | **核心文件（约5700行）**：Flask 后端 + PyWebView 桌面窗口，集成 Excel 解析、CDP 建副表、Coze 并行调用、台账写入、视频下载、**钉钉集成 UI** 全部功能 |
| `src/dingtalk_client.py` | **钉钉集成**：扫描产品段落、解析模式映射/需求数量、最近日期匹配、模板数据清洗与随机组合 |
| `src/excel_parser.py` | Excel 解析与清洗：表头识别、视觉组锁定抽取、比例小数还原、时长字段兜底 |
| `data/user_state.json` | 界面配置 + 已解析产品数据的持久化存储，刷新/重启不丢失 |
| `data/created_subtables.json` | 已建副表名记录（按 `base_token` 分桶），CDP 防撞名预判 + 自我学习 |
| `data/output/下载视频/{日期}/` | 视频下载目录，每个副表存于 `{日期}/{副表名}/`，打包为同层级 `{副表名}.zip` |
| `data/input/parsed/{日期}/` | Excel 解析后的各产品 JSON，命名格式：`日期_组名_产品名.json` |
| `src/ledger.py` | 台账 Excel 的创建、读取、更新【待开发，目前台账用飞书 Bitable】 |
| `src/scheduler.py` | 主流程调度【待开发】 |

## 四、关键设计决策

### 4.1 飞书认证方式（重要）
- **旧方式（已废弃）**：`user_access_token`（从 API Explorer 复制）→ 容易遇到 `99991672` 权限不足
- **新方式（当前）**：飞书自建应用 App ID + App Secret → 换取 `tenant_access_token`
- 应用需在飞书开放平台开通 `base:record:create` 权限，且手动加入目标 Bitable（"可编辑"）

### 4.2 Coze 工作流调用参数
- **入参**：`input`（数据数组，每个元素是一行 `//` 分隔的字符串）、`name`（副表名）
- **出参**：Coze API 返回 `data.debug_url`，记录工作流执行页面链接
- **并发策略**：ThreadPoolExecutor（max_workers=5），最多5个产品同时调用
- **超时**：每次调用 120 秒（用系统 curl，不受 Python LibreSSL 限制）

### 4.3 网络层统一用系统 curl
- 原因：macOS 系统 Python 的 `urllib` / `requests` 受 LibreSSL 版本限制，HTTPS 握手可能卡死
- 方案：`_curl_post` / `_curl_get` 函数封装 `subprocess.run(['curl', ...])`
- 所有 API 调用（Coze、飞书 OAuth、飞书 Bitable）全部走这两个函数

### 4.4 CDP 建副表与防撞名
- 通过 Chrome DevTools Protocol 操控本地 Chrome 浏览器，自动在飞书多维表格建表并写数据
- Base Token 从界面配置中读取
- Step1 失败时只打 ⚠️ 警告，**不中断** Step2 Coze 调用
- **三道防线防撞名**：
  1. **本地记录预判**：`data/created_subtables.json` 按 `base_token` 分桶，CDP 操作前直接 bump 到未占用候选
  2. **UI 红字检测**：fill 后扫描页面 `已存在` 文本节点 + 可见性判断
  3. **按钮 disabled 兜底**：飞书"复制"按钮被禁用也视为撞名
- **自我学习**：UI 检测命中时通过 `on_dup_found` 回调把已占用名写入本地记录
- 最终名（如 `_3`）自动回写 `product["key"]`，Step2 Coze 调用同步使用

### 4.5 UI 状态持久化
- 配置项（Token、ID 等）和已解析产品数据统一存入 `data/user_state.json`
- 界面刷新/重启后自动还原，无需重新填写
- JS 中用 `autoSaveConfig()` 节流保存，`loadState()` 页面加载时恢复

### 4.6 视频下载工具（附加功能）
- 从飞书多维表格拉取副表视频 URL，一键批量下载到本地
- **目录结构**：`data/output/下载视频/{任务日期}/{副表名}/`，视频文件命名 `{副表名}_{行号}.mp4`
- 打包：下载完成后自动 zip 到同层级 `{副表名}.zip`，可选删除原文件夹
- 飞书数据解析：调用内部 `clientvars` 接口，响应为 gzip+base64 压缩

### 4.7 钉钉集成模块
- 从钉钉 doc 读取**汇总表**（`ai数量需求汇总`）和各**产品 sheet**
- **副表命名规则**：`[日期]_[组名]_[产品名]_[序号]`（如 `5.22_何欢组_予香琳喷雾_1`），撞名自动递增后缀
- **模式1**：`find_recent_dates(avail, target, n=1)` — 从模板可用日期中找 ≤ 任务日期最近的 1 个日期，用其数据生成全部条数
- **模式2**：`find_recent_dates(avail, target, n=2)` — 取最近 2 个日期，各占一半条数混用
- **需求数量自动填入**：`parse_demand_map` 解析汇总表「需求数量」列，前端三级按钮（总/组/产品）一键填入或清除
- **数据分隔符**：` // `（空格+双斜杠+空格），字段间统一分隔
