## 一、当前实际流程（全部在 app_server.py 界面完成）

```
第一步：上传 Excel
  打开平台：python3 src/app_server.py
  在界面上传甲方 Excel
  → 自动解析，展示：日期 → 组名 → 产品（每个产品可勾选）

第二步：填写配置（首次填写后自动保存）
  Coze Token / 工作流ID
  飞书 App ID / App Secret
  飞书 Base Token（副表所在的 Bitable）

第三步：选择产品，点击「⚡ 确认，开始执行」
  系统自动串行执行三步（日志实时显示在界面）：

  Step 1/3 🗂️ 飞书 CDP 建副表
    ├─ 通过 CDP 操控本地 Chrome 打开飞书
    ├─ 三道防线防撞名（详见第五节）：本地记录预判 → UI 红字检测 → 按钮 disabled 兜底
    ├─ 自动创建副表，写入该产品所有数据行；撞名时自动 bump 后缀（_1 → _2）
    ├─ 最终名回写至 product["key"]，同步给 Step2 Coze 工作流
    └─ 失败只显示 ⚠️ 警告，不中断后续

  Step 2/3 🤖 并行触发 Coze 工作流
    ├─ ThreadPoolExecutor（max_workers=5）并行调用
    ├─ 入参：input（数据数组）、name（副表名称，如 5.16_刘原原组_娇茵舒凝胶）
    ├─ 出参：debug_url（工作流执行追踪链接）
    └─ 日志显示完整请求参数 + 响应体，方便调试

  Step 3/3 📋 写入飞书台账
    ├─ App ID + Secret → tenant_access_token
    ├─ 调用 batch_create 写入台账表
    ├─ 台账：PSgJbhr21aHRGosg4uHclshgnDe / tblZrh3e6CWvvSwp
    └─ 字段：任务名称（日期_组名_产品）/ debugurl（Coze 返回）
```

## 二、数据文件路径规范

| 路径 | 说明 |
|------|------|
| `data/甲方excel/` | 甲方原始 Excel 存放目录 |
| `data/input/parsed/{日期}/` | 界面解析后的各产品 JSON，命名：`日期_组名_产品名.json` |
| `data/user_state.json` | UI 状态持久化：配置 + 已解析产品数据 |
| `data/created_subtables.json` | CDP 已建副表名记录（按 `base_token` 分桶），用于防撞名预判 |
| `data/output/` | 下载的视频、生成的审核 Excel 等成品 |

## 三、Excel 解析规则（界面上传时自动执行）

1. **识别产品块**：每个产品以「产品名」行开头，后跟「日期//人物a//...」列头行，再跟数据行
2. **输出 JSON**：`[产品名, 列头字符串, 数据行1, 数据行2, ...]`（每行为 `//` 分隔的字符串）
3. **命名格式**：`日期_组名_产品名.json`，存入 `data/input/parsed/{日期}/`

### 特殊副表处理

| 副表 | 特殊情况 | 处理方式 |
|------|----------|----------|
| 燕娜组 | 两个产品**左右并排**排列 | 拆分为两个 JSON 文件 |
| 薛梦琪组 | 两个产品**上下堆叠**排列 | 拆分为两个 JSON 文件 |
| ai数量需求汇总 | 非视频数据表 | 跳过，不处理 |

## 四、Coze 工作流调用规范

### 当前工作流（视频生成）
- **入参 `input`**：数据数组，每个元素为 `//` 分隔的字符串（一行数据）
- **入参 `name`**：副表名称，格式 `日期_组名_产品名`（如 `5.16_刘原原组_娇茵舒凝胶`）
- **出参**：`data.debug_url`（工作流执行页面链接，用于追踪进度）
- **Coze API 接口**：`POST https://api.coze.cn/v1/workflow/run`
- **并发策略**：最多5个产品同时调用，避免接口限流

### 工作流返回示例
```json
{
  "code": 0,
  "msg": "success",
  "data": {
    "execute_id": "7641821250788458537",
    "debug_url": "https://www.coze.cn/work_flow?execute_id=...&workflow_id=..."
  }
}
```

### 错误处理
| 错误类型 | 处理方式 |
|----------|----------|
| CDP 建副表失败 | ⚠️ 警告，继续执行 Coze |
| Coze 返回非0 code | 记录错误信息，该产品标记失败，其他产品继续 |
| 调用超时（>120s） | 标记失败，记录日志 |
| 台账写入 91403 | 检查应用是否加入 Bitable 并设为"可编辑" |
| 台账写入 99991672 | 检查应用是否开通 `base:record:create` 权限 |

## 五、副表名防撞机制（CDP 三道防线）

历史上同一产品多次执行会生成同名副表（如 `5.22_何欢组_予香琳喷雾_1`），手工创建过的也会撞。
当前用「本地记录 + UI 检测 + 按钮兜底」三层叠加，确保任何情况下都能找到一个未被占用的名字。

### 5.1 防线分布

| 防线 | 触发时机 | 实现 |
|------|----------|------|
| ① 本地记录预判 | CDP 还没开始操作 | `_resolve_unique_subtable_name(base_token, requested)` 查 `data/created_subtables.json`,直接 bump 到未占用候选 |
| ② "已存在"红字检测 | fill 名字后 500ms | 扫描页面所有 `已存在` 文本节点 + 可见性判断 |
| ③ "复制"按钮 disabled 检测 | 同上 | `confirm_btn.is_disabled()` 兜底,防止飞书改了红字样式时漏判 |

### 5.2 自我学习

② 或 ③ 命中时,通过 `on_dup_found` 回调把**当前被占用的候选名**一并写入 `data/created_subtables.json`。
所以第一次跑会通过 UI 学到所有历史占用名,后续完全靠 ① 一次预判搞定。

```
第一次跑 5.22_何欢组_予香琳喷雾_1(本地记录空)
  ├─ ① 预判:不动
  ├─ fill _1 → ② 红字命中 → 记录 _1
  ├─ fill _2 → ✅ → 记录 _2 → 完成

第二次跑(本地记录 [_1, _2])
  ├─ ① 预判:requested=_1 → 直接 bump 到 _3
  └─ fill _3 → ✅ → 记录 _3 → 完成
```

### 5.3 名字同步给 Coze

撞名后改名的最终名(如 `_3`)会立即写回 `product["key"]`,Step2 触发 Coze 时
`parameters.name` 也是这个新名,保证副表数据和工作流入参一致,不会"建在 _3 但查 _1"。

## 六、时长字段(2026-05 表格新增)

| 项目 | 说明 |
|------|------|
| 表头识别 | `_classify_columns()` 将「时长」并入视觉组,与人物/场景/道具/动作/镜头**整组锁定**抽取 |
| 默认值 | `_DEFAULT_DURATION = '10'`,定义在 `src/excel_parser.py` |
| 兜底规则 | 空 / `/` 占位 / 非纯数字(如图片 dict 转的 JSON 串)→ 全部回退为 `'10'` |
| 合法值 | 正则 `^\d+(?:\.\d+)?$` 命中的原样保留(常见 `10`、`15`、`5.5`) |

## 七、比例列时间小数还原(Excel/钉钉历史问题)

甲方在 Excel 输入「9:16」时,部分单元格被自动识别为时间,底层存成 0~1 浮点小数:

| 原始 float | 还原后 |
|-----------|--------|
| `0.386111111111111` (= 9×60+16 分钟 / 1440) | `9:16` |
| `0.6729166666667`  (= 16×60+9 分钟 / 1440) | `16:9` |
| `0.16875`          (= 4×60+3 分钟 / 1440)  | `4:3` |

在 `_clean_cell` 中按启发式条件(`0 < cell < 1` 且 `cell × 1440` ≈ 整数,误差 ≤ 0.05 分钟,
且 `len(repr(cell)) > 5` 排除人为输入的 `0.5` / `0.25`)还原为标准比例字符串,
避免传给 Coze 后被识别为 `1:1` 兜底。
